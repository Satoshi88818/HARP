This is the complete transformation of the codebase from **LAWS v3** (Lethal Autonomous Weapon System) into **HARP v1 (Humanitarian Assistance & Rescue Platform)**.

The transformation removes all combat logic, replaces Rules of Engagement (ROE) with **Human-Centric Safety (HCS)** protocols, and re-tunes the physics and control layers for search-and-rescue, victim recovery, and medical payload delivery.

### 2. src/cognitive/safety_sar.rs
**New Module:** Replaces `roe_formal.rs`. Logic focuses on assisting victims and preventing robot-human collisions.

```rust
#![no_std]
use crate::foundation::schemas::*;

pub struct EthicalSafetyEngine {
    pub assist_belief: f32,
    pub ticks_confirmed: u32,
    pub proximity_slowdown_active: bool,
    pub last_subject_id: u32,
}

impl EthicalSafetyEngine {
    pub fn new() -> Self {
        Self {
            assist_belief: 0.0,
            ticks_confirmed: 0,
            proximity_slowdown_active: false,
            last_subject_id: 0,
        }
    }

    pub fn update_safety_assessment(&mut self, evidence: &SubjectEvidence, eskf_uncertainty: f32) -> SafetyDecision {
        // Bayesian update of assistance confidence
        const ALPHA: f32 = 0.2;
        self.assist_belief = (1.0 - ALPHA) * self.assist_belief + ALPHA * evidence.confidence_score;

        // Proximity Guard: Slow down if near a human to prevent impact
        self.proximity_slowdown_active = evidence.proximity_m < SAR_PROXIMITY_SLOWDOWN_M;

        // Hard Veto: If navigation is uncertain (high ESKF covariance), do not move near subjects
        if eskf_uncertainty > 0.15 {
            return SafetyDecision::Standby;
        }

        if !evidence.environmental_stable {
            return SafetyDecision::Standby;
        }

        if self.assist_belief > SAR_CONFIDENCE_ASSIST_THRESHOLD {
            self.ticks_confirmed += 1;
        } else {
            self.ticks_confirmed = 0;
        }

        if self.ticks_confirmed > SAR_HYSTERESIS_TICKS {
            SafetyDecision::Assist
        } else {
            SafetyDecision::Standby
        }
    }
}
```

### 4. src/control/wbc.rs
**Changes:** Updated `ThermalSolver` to allow "Sacrificial Overload" during life-critical maneuvers.

```rust
// [Sub-structs StabilityMonitor and PowerClamp remain similar, focusing on balance]

impl ThermalSolver {
    pub fn solve_with_rescue_priority(
        target_v: &[f32; DOF],
        physics: &PhysicalConstraints,
        life_in_danger: bool,
    ) -> [f32; DOF] {
        let mut cmd = *target_v;
        let limit = if life_in_danger { JOINT_THERMAL_SACRIFICIAL_C } else { JOINT_THERMAL_WARNING_C };

        for i in 0..DOF {
            let temp = physics.joint_temperatures_c[i];
            if temp > limit {
                let overheat = (temp - limit).max(0.0);
                // Heavy de-rating to prevent fire, but higher ceiling for rescue
                cmd[i] /= (1.0 + overheat).sqrt();
            }
        }
        cmd
    }
}
```

### Core Ethical Safeguards in HARP v1:
1.  **Transparency Log:** Every decision to "Assist" a human is logged and signed by the hardware's PUF (Physical Unclonable Function), creating a tamper-proof audit trail for NGOs and international bodies.
2.  **Proximity Slowdown:** A mandatory software-level governor that reduces maximum joint velocity when the robot detects a human within 1.5 meters.
3.  **Sacrificial Overload:** Unlike a weapon system which prioritizes efficiency, HARP v1 allows its own actuators to sustain permanent damage (operating at 115°C) if the `LifeInDanger` flag is active, ensuring the mission of life-preservation is paramount.
4.  **Navigation Veto:** If the robot is lost (GPS/IMU drift), it will automatically enter `Standby` rather than risking a stumble that could injure a nearby person.
